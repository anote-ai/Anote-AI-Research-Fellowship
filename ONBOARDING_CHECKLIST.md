# Anote AI Research Fellowship — Onboarding Checklist

**Summer 2026 | PM-Owned**

This checklist is for the PM to run through for every incoming fellow. The goal is that by June 1 (orientation), every fellow is fully set up and can hit the ground running on June 3.

---

## Timeline

| Phase | When | Owner |
|-------|------|-------|
| Offers sent & accepted | ~4 weeks before Jun 1 | Natan + PM |
| Tools provisioned | ~1 week before Jun 1 | PM |
| Welcome packet sent | ~3 days before Jun 1 | PM |
| Orientation | Jun 1, 6–8 PM EST | Natan + PM |
| Research & product assignments confirmed | By Jun 3 standup | PM |

---

## Phase 1 — After Offer Acceptance

**For each fellow:**

- [ ] Collect full name, email, GitHub username, LinkedIn, school, degree & year, and location
- [ ] Add fellow to [`FELLOW_ROSTER.md`](./FELLOW_ROSTER.md)
- [ ] Send a welcome email confirming their spot and next steps
- [ ] Add to the fellows Slack workspace / channel
- [ ] Add to all recurring calendar invites:
  - [ ] Orientation — Mon Jun 1, 6:00–8:00 PM EST
  - [ ] AI Research Standup — Every Tuesday, 5:00–5:30 PM EST
  - [ ] AI Product Standup — Every Thursday, 5:00–5:30 PM EST
  - [ ] Friday Fellowship Talks — Every Friday, 1:00–2:00 PM EST
  - [ ] Midpoint Check-ins — Wed Jul 1, 5:00–10:00 PM EST (assign slot)
  - [ ] Final Presentations — Wed Aug 5, 5:00–10:00 PM EST (assign slot)
  - [ ] Anote Summer Party — Thu Aug 7, 7:00–10:00 PM EST

---

## Phase 2 — Tools & Access Provisioning

**For each fellow, complete before June 1:**

### GitHub
- [ ] Add to the [anote-ai GitHub organization](https://github.com/orgs/anote-ai)
- [ ] Add to their assigned product repo with write access
- [ ] Add to the [anote-ai/Research](https://github.com/anote-ai/Research) repo
- [ ] Add to this fellowship repo ([anote-ai/Anote-AI-Research-Fellowship](https://github.com/anote-ai/Anote-AI-Research-Fellowship))
- [ ] Confirm they can clone and push to their repos

### Claude Code / Codex
- [ ] Provision Claude Code access (or Codex, per Natan's preference)
- [ ] Send setup instructions and confirm they can run it locally
- [ ] Share any internal documentation or prompt guidelines Anote uses

### arXiv
- [ ] Remind fellows to create an arXiv account if they don't have one
- [ ] Confirm they understand the submission process (endorse if needed for first-time submitters)

### Slides & Templates
- [ ] Share the [Anote Google Slides Template](https://docs.google.com/presentation/d/1x4E2dwBnDImHjMOMaI8lLWcRNjVWlnsYEJ3WOwlhKuA/edit?usp=sharing) with view/copy access
- [ ] Remind fellows to use Moda or Gamma for presentation formatting, in the Anote theme

---

## Phase 3 — Welcome Packet

Send a welcome email 2–3 days before orientation containing:

- [ ] Fellowship overview and what to expect on June 1
- [ ] Link to this repo and all markdown docs
- [ ] Zoom link for virtual sessions and WeWork address for in-person Fridays
- [ ] Preliminary research track options — ask them to come to orientation with a top 2–3 preferences
- [ ] Preliminary product list — ask them to come with preferences
- [ ] Link to the [Anote Google Slides Template](https://docs.google.com/presentation/d/1x4E2dwBnDImHjMOMaI8lLWcRNjVWlnsYEJ3WOwlhKuA/edit?usp=sharing)
- [ ] Confirmation of their July 1 check-in slot and August 5 presentation slot
- [ ] Contact info for the PM for any pre-orientation questions

## Phase 5 — Post-Orientation (By June 3 Standup)

- [ ] Finalize and publish all research track assignments in [`RESEARCH_FELLOW_MATCHING.md`](./RESEARCH_FELLOW_MATCHING.md)
- [ ] Finalize and publish all product assignments in [`PRODUCT_FELLOW_MATCHING.md`](./PRODUCT_FELLOW_MATCHING.md)
- [ ] Update [`FELLOW_ROSTER.md`](./FELLOW_ROSTER.md) with confirmed assignments
- [ ] Tick off all access items in the Fellow Roster access checklist
- [ ] Confirm all fellows are in Slack and have responded
- [ ] Follow up with any fellow who hasn't confirmed access to their repos or tools

---

## PM Notes

- If a fellow needs arXiv endorsement (required for first-time submitters in some categories), identify this early — endorsement takes time and should not be left to August.
- If a fellow is fully remote, confirm their time zone and ensure calendar invites reflect EST. Check that the Zoom link works for them before June 1.
- Any fellow without Claude Code access by June 3 is a blocker — escalate to Natan immediately.
